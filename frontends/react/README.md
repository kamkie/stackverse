# Frontend — React

React 19 + TypeScript (strict), Vite, react-router, TanStack Query. Plain CSS
from the shared design package ([spec/design/](../../spec/design)), imported
verbatim — this app has no stylesheets of its own. All user-facing text comes
from `GET /api/v1/messages/bundle` at runtime (no i18n library, no build-time
resources); keys the bundle doesn't contain render as their last segment, so
admins can translate them later through the Messages screen without a deploy.

## Commands

```sh
pnpm install
pnpm dev              # dev server on :5173, API mocked in-browser (MSW)
VITE_API_MOCK=false pnpm dev   # proxy /api and /auth to a gateway on :8000
pnpm test             # vitest + Testing Library against the same MSW handlers
pnpm build            # type-check + static production bundle in dist/
pnpm generate:api     # regenerate src/api/schema.ts from spec/openapi.yaml
```

## API types & mocks are derived from the contract

- `src/api/schema.ts` is generated by openapi-typescript from
  [spec/openapi.yaml](../../spec/openapi.yaml) (committed; regenerate when the
  spec changes). The typed client (`openapi-fetch`) and the MSW handlers
  (`openapi-msw`) both compile against it, so handler or call-site drift from
  the contract fails the build.
- Mock message content is imported from [spec/messages/](../../spec/messages)
  — the same seed files backends import on startup.

## Mocked auth in dev

`/auth/*` is the gateway's surface (not in the OpenAPI spec) and login is a
full-page navigation, which a service worker cannot answer — in mock mode the
dev server serves the SPA for `/auth/login` and the mock bootstrap completes
the "OIDC" dance. Pick the identity before clicking *Log in*:

```js
localStorage.setItem("stackverse.mock.login-as", "demo");      // no roles
localStorage.setItem("stackverse.mock.login-as", "moderator"); // reports, dashboard
localStorage.setItem("stackverse.mock.login-as", "admin");     // full backoffice
```

## Production

`npm run build` emits plain static files (`dist/`) servable by any gateway;
there is no server-side rendering and no runtime configuration — all API
calls are relative (`/api/...`, `/auth/...`) and carry the session cookie.
